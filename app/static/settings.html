NOX alarm system
Receive inputs, area status and alarms from a NOX (Securiton) panel via Modbus TCP and/or TIO ASCII. Area arm/disarm requires the NOX integration manual and will be added next.
Modbus TCP — read inputs/areas
Polls the NOX panel for input status registers (open / sabotage / deactivated / alarm).
Discover inputs
Scan a range of NOX module IDs and list every defined input. Useful for setup — bulk-add the ones you want to monitor.
Discover area register block
Raw register sweep, no decoding. Use to hunt for the block where NOX exposes area state (one register per area, decimal value). Pick a preset or set a custom range.
Inputs to monitor
Register address = module ID × 10 + input index. e.g. module 1001 input 1 → 10011.
No inputs configured.
Areas to monitor
Register address = area ID. State 1 = disarmed (Frakoblet), 5 = armed (Tilkoblet), others reported as raw code.
No areas configured.
Test area write (Phase 2 verification)
Writes via FC16 (the only documented NOX write FC). Reads back rapidly to capture transient failure bits if the panel rejects the write. Useful codes: 1 (disarm), 5 (arm) — codes 2/3/4 are transitional states the panel manages itself during an arm/disarm sequence; code 6 is auto-only.
TIO ASCII — push events
Listen for pipe-delimited messages (INP… / AREA…) from a NOX TIO virtual output. Configure the NOX TIO module to push to this host on the listen port.
